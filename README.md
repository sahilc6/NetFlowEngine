# DPI Engine (Java) - Deep Packet Inspection System

NetFlowEngine is a Java-based Deep Packet Inspection (DPI) system. It reads network packet captures (PCAP files), inspects the contents of the packets at a deep level (such as extracting SNI from TLS handshakes or Host headers from HTTP requests), and filters traffic based on configurable blocking rules.

---

## 1. Project Overview

The DPI Engine processes raw PCAP files generated by tools like Wireshark or tcpdump. It parses the packet headers (Ethernet, IPv4, TCP/UDP) and extracts domain names using Deep Packet Inspection on the application layer payload.

### Key Features
- **PCAP Parsing**: Directly reads and parses binary PCAP files.
- **Protocol Decoding**: Understands Ethernet, IPv4, TCP, and UDP headers.
- **Deep Packet Inspection (DPI)**:
  - Extracts Server Name Indication (SNI) from TLS (HTTPS) Client Hello packets.
  - Extracts Host headers from HTTP requests.
- **Flow Tracking**: Keeps track of TCP/UDP flows using the 5-tuple (Source IP, Dest IP, Source Port, Dest Port, Protocol).
- **Traffic Blocking**: Allows blocking traffic based on Source IP, Application type, or Domain name (substring match).
- **PCAP Output**: Generates a filtered PCAP file containing only the unblocked packets.

## 3. How It Works

1. **Reading**: The `PcapReader` reads the 24-byte PCAP global header, followed by each 16-byte packet header and the packet payload.
2. **Parsing**: The `PacketParser` looks at the raw bytes, stepping through the Ethernet, IPv4, and TCP/UDP layers to extract source/destination IPs and ports.
3. **Tracking**: The `Main` class maps each packet to a specific connection ("flow") using its 5-tuple.
4. **Inspection**: If the traffic is HTTPS (port 443) or HTTP (port 80), the `SniExtractor` examines the TCP payload. It searches for the TLS Client Hello to extract the SNI, or parses HTTP headers to find the Host.
5. **Filtering**: The blocking rules are evaluated against the connection's source IP, extracted application type, and domain name. If the flow is blocked, all its packets are dropped.
6. **Writing**: Forwarded packets are written to a new PCAP file (`output.pcap`) via standard I/O.

## 4. Building and Running

### Requirements
- **Java 17** or higher
- **Maven 3.6+**

### Building the Project
Compile and package the project into an executable JAR using Maven:
```bash
mvn clean package
```
This will produce a JAR file in the `target` directory (e.g., `dpi-engine-1.0-SNAPSHOT-jar-with-dependencies.jar`).

### Running the Engine

You can run the engine in two ways:

#### Option A: Running Directly with Maven (Development)
You can compile and run the engine directly using Maven without packaging a JAR.

*On PowerShell / Windows Command Prompt:*
```powershell
mvn exec:java "-Dexec.mainClass=com.netflowengine.Main" "-Dexec.args=test_dpi.pcap output.pcap [options]"
```

*On Linux / macOS / Git Bash:*
```bash
mvn exec:java -Dexec.mainClass="com.netflowengine.Main" -Dexec.args="test_dpi.pcap output.pcap [options]"
```

#### Option B: Running the Executable JAR (Production)
First, package the project:
```bash
mvn clean package
```
Then run the generated JAR from the `target` directory:
```bash
java -jar target/dpi-engine-1.0-SNAPSHOT-jar-with-dependencies.jar <input.pcap> <output.pcap> [options]
```

#### Command Options:
- `--block-ip <ip>`: Block traffic originating from the specified source IP.
- `--block-app <app>`: Block a specific application (e.g., YouTube, Facebook, Netflix).
- `--block-domain <dom>`: Block any domain that contains this substring.

#### Examples:
**Running with Maven (Windows/PowerShell):**
```powershell
mvn exec:java "-Dexec.mainClass=com.netflowengine.Main" "-Dexec.args=test_dpi.pcap output.pcap --block-app YouTube --block-ip 192.168.1.50"
```

**Running with JAR:**
```bash
java -jar target/dpi-engine-1.0-SNAPSHOT-jar-with-dependencies.jar test_dpi.pcap output.pcap --block-app YouTube --block-ip 192.168.1.50 --block-domain facebook.com
```

## 5. Output Report

After processing a PCAP file, the system prints a comprehensive report summarizing:
- Total packets processed, forwarded, and dropped.
- The number of active flows detected.
- An application breakdown showing the percentage of traffic belonging to known applications.
- A list of unique domains/SNIs identified during deep packet inspection.
